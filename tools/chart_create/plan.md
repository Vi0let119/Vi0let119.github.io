# 图表生成工具 — 完整开发计划

## 项目概述

基于 AI 交互和 D3/ECharts 库生成图表的 Web 工具，集成到现有侧边栏「工具箱」板块。

| 维度 | 决策 |
|------|------|
| 后端 | Python Flask（API 代理 + Excel 解析） |
| 默认 AI | DeepSeek（OpenAI 兼容接口） |
| 自提供 API | 支持 DeepSeek / 千问 / OpenAI / Gemini 等主流平台 |
| 图表库 | ECharts 为主（交互性好），D3 为辅（高度自定义场景） |
| 图表类型 | 柱状图、饼图、折线图、散点图 |
| 默认风格 | 简约风（干净配色、适当留白、清晰标注） |
| 输出格式 | 独立 HTML 文件，内嵌 CDN 引用，打开即显示交互式图表 |

---

## 页面结构（3 个页面）

### 页面 1：前导页 `chart_landing.html`
- 说明极速版和 AI 交互版的适用场景
- 两个入口卡片/按钮，分别进入两个版本
- 极速版描述：数据规范、需求明确、快速出图（秒级）
- AI 交互版描述：数据复杂、格式不规范、需要 AI 分析引导

### 页面 2：极速版 `chart_quick.html`
- 纯前端处理，无需后端
- 表单式操作：图表类型 → 数据输入 → 坐标轴名称 → 标题 → 预览 → 下载
- 实时预览图表
- 一键下载独立 HTML 文件

### 页面 3：AI 交互版 `chart_ai.html`
- 聊天式界面，连接后端 AI 代理
- 引导流程：理解数据 → 明确需求 → 生成图表 → 迭代修改
- 支持文件上传（CSV/Excel/文本）
- 支持切换 API（默认站点 DeepSeek / 用户自提供 key）
- 右侧图表预览 + 下载按钮

## 极速版详细设计

### 表单字段
1. **图表类型**（下拉选择）：柱状图 / 饼图 / 折线图 / 散点图
2. **数据输入方式**（Tab 切换）：
   - 文本输入：粘贴数据，支持空格/逗号/Tab 分隔
   - 文件上传：.txt / .csv / .xlsx（SheetJS 客户端解析）
3. **数据格式提示**（醒目的提示框）：两列数据，第一列 X 轴，第二列 Y 轴
4. **第一行为表头**（复选框）：勾选后将第一行作为表头，自动提取坐标轴名称
5. **X 轴名称**（文本框）：如"月份"
6. **Y 轴名称**（文本框）：如"销售额（万元）"
7. **图表标题**（文本框）
8. **预览区**：实时 ECharts 渲染
9. **下载按钮**：生成独立 HTML 文件并触发下载

### 数据处理流程（客户端 JS）
```
1. 读取输入（文本框内容 / 上传文件内容）
2. 检测分隔符（空格 / 逗号 / Tab / 混合）
3. 按行分割，每行解析为两列
4. 如果勾选"第一行为表头" → 第一行不作为数据，提取为默认坐标轴名称
5. 自动判断数据类型（数值 / 字符串）
6. 过滤空行和无效行
7. 生成 ECharts option → 渲染到预览区
8. "下载"时构建完整 HTML → Blob 下载
```

### 技术要点
- **Excel 解析**：CDN 引入 SheetJS (`xlsx`)，读取 `.xlsx/.xls` 文件
- **图表渲染**：CDN 引入 ECharts，生成 option 配置
- **下载**：HTML 模板拼接 ECharts option → `new Blob()` → `<a download>`
- **文件不落盘**：所有操作在浏览器内存完成

## AI 交互版详细设计

### 聊天界面布局
- **左侧（对话区）**：聊天消息列表 + 底部输入框 + 文件上传按钮
- **右侧（预览区）**：图表实时预览 + 下载按钮
- **顶部（设置栏）**：API 选择（默认站点 / 自提供 key → 平台下拉 → key 输入框）

### 引导流程（后端 grill-me 风格追问）

**第一阶段 — 理解数据**：
- 用户上传文件或粘贴原始数据
- AI 分析：列数、每列数据类型（数值/分类/时间）、数据范围、缺失情况
- 如有歧义，追问确认（例如"检测到第2列和第3列都是数值，哪一列作为 Y 轴？"）

**第二阶段 — 明确需求**：
- AI 询问图表目的：对比 / 趋势 / 占比 / 分布关系
- 结合数据结构推荐图表类型，用户确认或修改
- 确认图表标题、坐标轴标注等细节

**第三阶段 — 生成图表**：
- AI 返回 ECharts option（JSON 格式）
- 后端解析并注入 HTML 模板，返回给前端
- 右侧预览区渲染

**第四阶段 — 迭代修改**：
- 用户提出修改意见（改颜色、调范围、换类型、修正数据映射）
- AI 调整 option 并重新生成预览

### API 代理设计
- 后端统一接口 `/api/chat`，前端不区分具体平台
- 后端根据用户选择的平台，路由到对应 API
- 支持的平台：

| 平台 | API 端点 | 兼容性 |
|------|----------|--------|
| DeepSeek | `https://api.deepseek.com/v1` | OpenAI 兼容（默认） |
| 千问 (Qwen) | `https://dashscope.aliyuncs.com/compatible-mode/v1` | OpenAI 兼容 |
| OpenAI | `https://api.openai.com/v1` | 标准 |
| Gemini | `https://generativelanguage.googleapis.com/v1beta` | Google 格式 |

- 用户自提供 key 仅存在服务端 session 中，不写入磁盘

### AI System Prompt 核心要点
- 角色：图表生成专家助手
- 能力：理解任意格式的数据，推荐合适图表，生成 ECharts option
- 风格约束：简约风，柔和低饱和度配色，清晰标注
- 交互规则：先确认理解再生成，不确定时主动追问
- 输出格式：ECharts option JSON + 简要说明

## 后端自动 Skills（用户无需选择，自动生效）

| Skill | 作用 | 实现方式 |
|-------|------|----------|
| **数据识别** | 自动判断列类型（数值/分类/时间）、检测列数、识别分隔符 | Python 预处理函数 |
| **图表推荐** | 根据数据结构推荐最合适的图表类型 | 规则引擎 + AI 辅助 |
| **配色方案** | 简约风默认配色，根据数据系列数量自动调色 | 预设调色板数组 |
| **数据清洗** | 自动过滤空行、处理缺失值、去除非数值噪声 | Python 清洗函数 |
| **追问引导** | AI 交互版中 grill-me 风格的多轮确认对话 | System Prompt 策略 |
| **设计质量** | 输出图表遵循 frontend-design 美学原则 | HTML 模板 + AI 生成 |
